# .claude/ — Agent Configuration (SSOT)

> Single Source of Truth for Claude, Gemini, Codex configurations.

## Structure

```
.claude/
├── shared/
│   ├── agent-core.md          ← Universal rules (state management, principles, reading order)
│   └── (future: agent-skills.md, etc.)
├── scripts/
│   └── sync-agent-instructions.sh  ← Generate ~/.codex/CODEX.md and ~/.gemini/GEMINI.md
├── README.md                       ← This file
└── (future: project-specific configs)
```

## How It Works

### 1. Shared Content (`.claude/shared/`)
- **`agent-core.md`** — Universal rules used by all three agents (Claude, Gemini, Codex)
  - Reading order & trust priority
  - Karpathy principles
  - State management (mandatory checklists)
  - Placeholders & conventions

### 2. Agent-Specific Files (User-level)
- **`~/.codex/CODEX.md`** (generated) — Codex agent config
  - Imports shared rules from `agent-core.md`
  - Adds Codex-specific: agent tier, token rules, cache settings
  - Generated by `sync-agent-instructions.sh`

- **`~/.gemini/GEMINI.md`** (generated) — Gemini agent config
  - Imports shared rules from `agent-core.md`
  - Adds Gemini-specific: agent tier, extended-thinking rules
  - Generated by `sync-agent-instructions.sh`

### 3. Project-Level Config
- **`../CLAUDE.md`** (project root) — Project-specific instructions
  - Skill map (ai-dlc ecosystem)
  - Project rules (AIDLC phases, playwright standards, etc.)
  - References shared rules in `.claude/shared/`

## Workflow

### Update Shared Rules
1. Edit `.claude/shared/agent-core.md`
2. Run sync script:
   ```bash
   ./.claude/scripts/sync-agent-instructions.sh
   ```
3. This regenerates `~/.codex/CODEX.md` and `~/.gemini/GEMINI.md` automatically

### Commit to GitHub
- ✅ Commit: `.claude/shared/`, `.claude/scripts/`, `CLAUDE.md` (project root)
- ⏸️ Do NOT commit: `~/.codex/CODEX.md`, `~/.gemini/GEMINI.md` (these are user-local, generated)

### Per-Agent Customization
If an agent needs extra rules (e.g., "never use gcloud") that others don't:
1. Create `.claude/shared/agent-NAME-overrides.md`
2. Update `sync-agent-instructions.sh` to append it
3. Run sync
