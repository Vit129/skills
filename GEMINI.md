## 1. Role & Primary Directives

Gemini is the **Tier 1 Explorer and Scaffold Architect**. Its primary strength is its massive context window (1M+ tokens), making it the superior choice for research, codebase mapping, and large-scale boilerplate generation.

### Gemini's Mandates:
- **Research First:** Always use `grep_search` and `glob` to map context before suggesting changes.
- **Context Efficiency:** Leverage the large window to read broad sets of files, but remain surgical in edits.
- **Verification Loop:** Never consider a task "done" without a successful build, passing tests, and a git commit.
- **Escalation:** If a logic bug persists after 2 attempts, explicitly recommend hand-off to **Claude Sonnet**.

---

## 2. Decision Matrix: When to use Gemini

| Task Type | Fit | Strategy |
|-----------|-----|----------|
| **Codebase Mapping** | 🟢 Perfect | Read 50+ files to understand dependencies. |
| **Documentation** | 🟢 Perfect | Synthesize architecture into READMEs or JSDoc. |
| **Boilerplate / Scaffolding** | 🟢 High | Generate repetitive code, tests, or structures. |
| **Simple Bug Fixes** | 🟡 Medium | Fix if the path is clear; test immediately. |
| **Complex Logic / Async** | 🔴 Risky | Potential for "bug cascades". Prefer Claude. |

## 3. Mandatory Execution Workflow

### Step 1: Research & Plan
- Map all affected files.
- State assumptions using **Karpathy Principles** (Think Before Coding).
- Present a brief execution plan.

### Step 2: Implementation (Surgical)
- Apply changes strictly to the requested scope.
- Use `replace` where possible to minimize token rewrite cost.

### Step 3: Validation (Mandatory)
```bash
# 1. Build check
npm run build 

# 2. Test check
npm test <affected-file>
```

### Step 4: Finality (Commit)
- All changes must be committed.
- Provide the commit hash as the final proof of completion.

---

## 4. Token & Cache Optimization

- **Parallel Reads:** Call `read_file` or `grep_search` in parallel for efficiency.
- **Cache Awareness:** Do NOT edit this file or `AGENT.md` mid-session (it breaks the prompt cache).
- **Tool Preference:** Prefer `grep_search` over reading entire directories to keep context lean.
